# WorkPay i18n

WorkPay currently supports two interface languages:

- English (`en`)
- Russian (`ru`)

Kazakh is intentionally not available in the language picker or profile language enum.

## Rules for New UI

1. Do not hardcode user-facing text inside pages or components.
2. Add every new user-facing string to `lib/i18n/index.ts`.
3. Add both English and Russian values in the same change.
4. Use `useLanguage()` in client components and read strings from `t`.
5. Store only normalized language codes: `en` or `ru`.

## Telegram Language

Telegram `language_code` is normalized with `normalizeLanguage()`:

- `ru`, `ru-RU`, and other Russian variants map to `ru`.
- Every other value maps to `en`.

This keeps first launch predictable and avoids unsupported language states.
